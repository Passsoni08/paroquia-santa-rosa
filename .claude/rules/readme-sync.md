# Manter README.md atualizado

Sempre que você fizer alterações que afetem qualquer um dos itens abaixo, atualize o `README.md` na raiz do projeto:

- Novos comandos ou scripts (pyproject.toml, package.json)
- Mudanças em variáveis de ambiente (.env.example)
- Alterações no docker-compose.yml ou Dockerfiles
- Novas dependências relevantes
- Mudanças em portas, URLs ou endpoints
- Novos pré-requisitos de sistema
- Alterações no fluxo de setup ou deploy

**Não atualize o README para mudanças internas** que não afetam o setup ou uso do projeto (refatorações, correções de bugs, novos componentes, etc).
